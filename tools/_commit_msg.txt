fix: remove DB.CreateQuery to prevent native crash on context destroy (VP-SELECTCIV-DBCRASH-1)

DB.CreateQuery creates sqlite3_stmt* C++ objects as Lua userdata with native
finalizers. When UIManager:DequeuePopup + ContextPtr:SetHide destroy the
SelectCivilization Lua context, the engine can invalidate the DB handle before
the Lua GC calls those finalizers, causing an access violation (0xC0000005) in
CvGameDatabaseWin32FinalRelease.dll. No Lua pcall can intercept this.

Fix: remove all three prepared statements (_unitsQ, _buildingsQ, _improvementsQ),
_initQueries(), _appendUnique(), and the three DB-dependent sections (3/4/5) of
_buildRichLabel. Replace with a zero-DB approach: after VP's AddCivilizationEntry
returns, read tooltip text from the B1..B6 icon buttons VP already rendered
(ct["B" .. i]:GetToolTipString()). All DB access is gone from the FrontEnd context.
