Fix SelectCivilization not vocalized (VP-SELECT-CIV-2)

Root cause: CivVAccess_SelectCivilizationAccess (CVA DLC) calls
rebuildItems() at include-time, which queries Leaders.Description
via DB. VP has leaders where this field is NULL, causing
Text.key(nil) -> ConvertTextKey(nil) -> Lua error. The pcall in
our bridge caught the crash but BaseMenu.install never completed,
so no accessibility hook was installed.

Fix: replace the bridge in SelectCivilization.lua to call a new
VP-compat-specific wrapper instead of the CVA DLC wrapper.

- add src/vp-compat/UI/FrontEnd/CivVAccess_VP_SelectCivilizationAccess.lua
  monkeypatches AddCivilizationEntry and AddRandomCivilizationEntry to
  collect entries lazily at show-time (Title:GetText()), avoiding all
  DB queries at include-time. Wraps ShowHideHandler and InputHandler
  for Up/Down/Enter navigation with SpeechPipeline.speakInterrupt.
- update src/vp-compat/UI/FrontEnd/SelectCivilization.lua bridge to
  call CivVAccess_VP_SelectCivilizationAccess (MD5 5AFC69A8)
- update src/vp-compat/CivVAccess_VoxPopuli.modinfo: new MD5 for
  SelectCivilization.lua + new File entry for VP wrapper (MD5 846C9481)
- update tools/deploy.ps1: add both SelectCivilization files
- deploy to MODS/vp-compat verified (MD5 OK for all 3 changed files)

All [AUTO] checks in validate-vp-compat.ps1 still pass.
